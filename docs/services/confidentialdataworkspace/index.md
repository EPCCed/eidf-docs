# Overview

The Confidential Data Workspace (CDW) service is a specialised offering within the EIDF, designed to provide data owners with enhanced control over how data enters and leaves their project environment. The service is built on the same underlying EIDF Virtual Machine (VM) Service infrastructure with additional access controls, network isolation, and policy requirements to meet the needs of secure data handling.

The service currently has a mixture of hardware node types which host VMs of various flavours. These match the [virtual machine service](../virtualmachines/flavours.md) flavours.

These pages describe the service features specific to the CDW, and highlight differences from the general VM service where relevant.

## Data Processing Requirements

We have policies in place regarding the kinds of data that are permitted inside the EIDF, and we may require additional evidence to ensure compliance with EIDF and relevant regulatory policies before use of the CDW service.

For a full list of data types and their restrictions, please see the [EIDF Third Party Data Policy](https://edinburgh-international-data-facility.ed.ac.uk/about/policies/third-party-data).

Please see the [Confidential and sensitive data](https://edinburgh-international-data-facility.ed.ac.uk/confidential-and-sensitive-data) page for a full comparison between the EIDF Virtual Desktop service, the CDW, and EPCC's Safe Haven Services.

## Service Access

Users should have an EIDF account - [EIDF Accounts](../../access/project.md).

Project Leads will be able to have access to the CDW added to their project during the application process or through a [support request](https://portal.eidf.ac.uk/queries/submit).
